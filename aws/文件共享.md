## 问题 #260（纠错后）

某公司的**合规团队**需要将其**文件共享**迁移到 AWS。  
这些共享当前运行在 **Windows Server SMB 文件共享**上，并且使用一个**自管理的本地 Active Directory（AD）** 来控制对文件和文件夹的访问。

公司计划使用 **Amazon FSx for Windows File Server** 作为文件共享解决方案。  
公司必须确保：  
> **在迁移到 AWS 后，仍然可以使用本地 Active Directory 组来限制对 FSx for Windows 的 SMB 合规共享、文件夹和文件的访问权限。**

公司已经创建了一个 **FSx for Windows File Server 文件系统**。

哪种解决方案可以满足这些要求？

A. 创建一个 Active Directory 连接器以连接到 Active Directory。将 AD 组映射到 IAM 组以限制访问。  
B. 使用标签（Tag）控制访问，并将 AD 组映射到 IAM 组以限制访问。  
C. 创建一个直接链接到 FSx for Windows 的 IAM 服务关联角色以限制访问。  
D. **将文件系统加入 Active Directory 以限制访问。**

---

## 正确答案
**👉 D**

---

## 解析

### 考点
- **Amazon FSx for Windows File Server 的身份验证方式**
- **Windows SMB 权限模型**
- Active Directory 与 IAM 的职责边界

---

### 为什么选 D

- **Amazon FSx for Windows File Server 原生集成 Active Directory**
- 只要：
  - 将 FSx 文件系统 **加入现有的本地 Active Directory**
- 就可以：
  - 继续使用 **AD 用户和 AD 组**
  - 控制 **SMB 共享 / 文件夹 / 文件级别的 NTFS 权限**
